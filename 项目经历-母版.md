## 二、功能描述

实现一个支持多客户端同时在线聊天的 C/S 架构程序。

**客户端功能：**
- 通过 IP 地址 + 端口号连接服务器
- 设置用户名，连接后自动注册到服务器
- 私聊：向指定用户发送消息（对方单独收到）
- 群聊：向所有在线用户广播消息
- 实时接收并显示其他用户发来的消息
- 支持主动断开连接

**服务端功能：**
- 监听指定端口，接受多个客户端连接
- 维护在线用户表（用户名 → 连接 的映射）
- 消息路由：解析协议→查找目标→转发消息
- 私聊转发：只发给指定用户
- 群聊转发：发给所有用户（跳过发送者本人）
- 用户上线/下线广播通知
- 重复登录检测（同一用户名新连接踢掉旧连接）
- 服务器日志：记录所有连接、消息路由、异常事件

**三种消息输入方式（客户端）：**

| 输入方式 | 输入示例 | 实际发出 | 效果 |
|---------|---------|---------|------|
| 直接写协议格式 | `S|小红|你好` | `S|小红|你好` | 私聊给小红 |
| 直接写协议格式 | `T|大家好` | `T|大家好` | 群发所有人 |
| 简写格式 | `@小红 你好` | `S|小红|你好` | 私聊给小红 |
| 普通文本 | `大家好` | `T|大家好` | 群发所有人 |

### 3.2 服务端线程模型

采用 **「一客户端一线程」** 的线程池模型：

- **主线程**：负责 UI 刷新、QTcpServer 监听新连接、消息路由分发。所有 `QMap<用户名, ClientHandler*>` 操作仅在主线程执行，天然线程安全，无需加锁。
- **Worker 线程**（每个客户端一个）：负责该客户端的 socket 数据读写。`QTcpSocket` 对象通过 `moveToThread()` 移到 Worker 线程，确保 socket I/O 在所属线程的事件循环中执行。
- **跨线程通信**：主线程与 Worker 线程之间通过 `Qt::QueuedConnection` 信号/槽通信，Qt 框架保证线程安全。主线程通过 `QMetaObject::invokeMethod()` 调用 Worker 线程的 `writeMessage()` 槽函数来发送数据。

### 3.3 客户端线程模型

采用 **「UI 线程 + Socket Worker 线程」** 的双线程模型：

- **UI 线程（主线程）**：处理界面交互、消息格式化（`formatMessage()`）、消息展示。通过 `QMetaObject::invokeMethod()` 将连接/发送请求排队发送给 Worker 线程。
- **Socket Worker 线程**：拥有 `QTcpSocket` 对象的所有权，在独立事件循环中处理 socket 的 connect、read、write、disconnect 操作。通过信号 `dataReceived()` 将收到的数据发回 UI 线程。
- **线程安全**：所有 UI 操作在主线程，所有 socket 操作在 Worker 线程，通过 QueuedConnection 桥接。

## 四、通信协议设计

自定义应用层协议，基于纯文本，使用 `|` 作为字段分隔符：

| 协议类型 | 格式 | 含义 | 示例 |
|---------|------|------|------|
| 注册/登录 | `R|用户名` | 客户端连接后向服务器注册用户名 | `R|小明` |
| 单播(私聊) | `S|目标用户名|消息内容` | 向指定用户发送消息 | `S|小红|你好` |
| 组播(群发) | `T|消息内容` | 向所有在线用户群发消息 | `T|大家好啊` |

**协议解析逻辑（服务端）：**

```
接收到消息
  ├── 以 "R|" 开头 → 注册用户名 → 插入用户表 → 广播"XXX上线啦！"
  ├── 以 "S|" 开头 → 提取目标用户名和消息 → 查表 → 原样转发消息内容
  └── 以 "T|" 开头 → 提取消息 → 遍历用户表（跳过发送者）→ 原样群发消息
```

**消息格式化逻辑（客户端）：**

```
用户输入
  ├── 已是 "S|用户名|消息" → 原样发送（支持消息中带 | 字符）
  ├── 已是 "T|消息"       → 原样发送
  ├── "@用户名 消息"      → 自动转为 "S|用户名|消息"
  └── 其他文本            → 自动转为 "T|消息"
```

## 五、关键技术实现

### 5.1 服务端 accept + 线程分配

```cpp
// 每个客户端连接 → 创建独立线程 + 独立处理器
void Serve1::onNewConnection() {
    while (QTcpSocket *clientSocket = m_server->nextPendingConnection()) {
        QThread *thread = new QThread(this);
        // 1. 将 socket 移到工作线程（必须在 start() 前）
        clientSocket->setParent(nullptr);
        clientSocket->moveToThread(thread);
        // 2. 创建处理器并移到同线程
        ClientHandler *handler = new ClientHandler(clientSocket);
        handler->moveToThread(thread);
        // 3. 跨线程信号/槽连接（QueuedConnection）
        connect(handler, &ClientHandler::messageReceived,
                this, &Serve1::routeMessage, Qt::QueuedConnection);
        // 4. 启动工作线程事件循环
        thread->start();
    }
}
```

### 5.2 线程安全的跨线程写入

```cpp
// 主线程 → Worker 线程安全写数据
QMetaObject::invokeMethod(
    handler,              // 目标对象（在 Worker 线程中）
    "writeMessage",       // 槽函数名
    Qt::QueuedConnection,  // 排队连接：调用安全地排队到目标线程
    Q_ARG(QString, msg)   // 参数
);
```

### 5.3 线程安全的用户表管理

所有 `QMap<QString, ClientHandler*>` 的增删查操作都在主线程的 `routeMessage()` 中执行，借助 Qt 的 QueuedConnection 机制天然序列化，无需 `QMutex`。

### 5.4 客户端消息格式化与协议兼容

使用 `QString::section('|', ...)` 替代 `split('|')`，确保消息内容中包含 `|` 字符时协议不会被错误切分。

## 六、开发过程中发现并修复的 Bug

| 编号 | 问题描述 | 根因 | 修复方案 |
|------|---------|------|---------|
| BUG-1 | 两个客户端都收不到消息 | `QTcpServer::nextPendingConnection()` 返回的 socket 属于主线程，但只把 handler 移到了工作线程，socket 未移。跨线程操作 socket 导致 I/O 失败 | 在创建 handler 之前将 socket 通过 `setParent(nullptr)` + `moveToThread(thread)` 移到工作线程 |
| BUG-2 | 服务端退出时 handler 内存泄漏 | `connect(thread, finished, handler, deleteLater)` 无效：`finished` 信号发出时工作线程的事件循环已停止，`deleteLater` 投递的删除事件无法被处理 | 改为在 `onHandlerDisconnected()` 中先调用 `handler->deleteLater()`（事件循环仍在运行），再 `thread->quit()` → `wait()` → `delete thread` |
| BUG-3 | 编译错误 `ui::textEdit` | 手误将 `ui->textEdit` 写成 `ui::textEdit`（`->` vs `::`） | 修正为 `ui->textEdit` |
| BUG-4 | 服务端转发消息时附加了"XXX对你说"前缀，改变了原始协议行为 | 开发时为了方便添加了发送者前缀 | 改为原样转发消息内容，保持协议透明性 |

### 7.4 线程安全性验证

| 验证项 | 方法 | 结论 |
|--------|------|------|
| 用户表并发访问 | 主线程独占 `m_userMap` 的所有读写操作 | 串行化，无竞态条件 |
| 跨线程 socket 写入 | 统一通过 `QMetaObject::invokeMethod` + `Qt::QueuedConnection` | 写入操作排队到 socket 所属线程 |
| 线程生命周期 | 断连事件 → `deleteLater` 投递 → `quit` → `wait` → 删除 thread | 对象在正确时机释放 |
